# О проекте

## Общая информация

Данный проект посвящён анализу и предобработке датасета сетевого трафика **CSE-CIC-IDS2018**, который используется для задач обнаружения сетевых атак и разработки систем информационной безопасности.

Набор данных содержит информацию о сетевых потоках, включающих как нормальный трафик, так и различные типы атак. Каждый сетевой поток описывается набором числовых и категориальных признаков, отражающих характеристики соединения.

---

## Актуальность темы

С увеличением количества кибератак возрастает необходимость в автоматизированных системах обнаружения вторжений (IDS — Intrusion Detection Systems).

Ручной анализ сетевого трафика невозможен из-за его объема, поэтому применяются методы анализа данных и машинного обучения.

Данный проект направлен на изучение этапов подготовки данных для таких систем.

---

## Цель проекта

Целью работы является исследование структуры датасета сетевого трафика, выявление проблем в данных, их очистка и подготовка к дальнейшему использованию в моделях машинного обучения.

---

## Задачи проекта

В рамках проекта были поставлены следующие задачи:

- изучить структуру датасета CSE-CIC-IDS2018;
- проанализировать признаки сетевого трафика;
- объединить данные из нескольких файлов;
- выявить пропуски и аномалии;
- обработать значения NaN и inf;
- провести очистку данных;
- выполнить анализ распределения классов;
- построить визуализации;
- подготовить данные для последующего моделирования.

---

## Описание датасета

Датасет состоит из нескольких файлов, каждый из которых соответствует определённому дню сетевой активности.

В данных присутствуют:

- нормальный сетевой трафик;
- различные типы атак (DDoS, DoS, Brute Force и другие);
- более 80 признаков, описывающих сетевые потоки.

Основные признаки:

- Destination Port
- Protocol
- Flow Duration
- Total Fwd Packets
- Total Backward Packets
- Flow Bytes/s
- Flow Packets/s
- Packet Length Mean
- Active Mean
- Idle Mean
- Label

Признак **Label** используется как целевая переменная для определения типа трафика.

## Основные проблемы в данных

В процессе анализа были выявлены следующие проблемы:

### 1. Пропущенные значения (NaN)

В ряде признаков присутствовали пустые значения, которые возникли из-за особенностей формирования сетевых потоков.

### 2. Бесконечные значения (inf)

Некоторые вычисляемые признаки содержали значения бесконечности, возникающие при делении на ноль.

### 3. Ошибочные типы данных

Часть значений не соответствовала числовому формату и требовала преобразования.

### 4. Дисбаланс классов

Распределение данных показало:

- нормальный трафик: ~87%
- атаки: ~13%

Это может негативно влиять на качество моделей машинного обучения.

---

## Предобработка данных

Для очистки данных были выполнены следующие операции:

- замена `inf` и `-inf` на NaN;
- преобразование признаков в числовой формат;
- удаление строк с пропусками;
- проверка корректности данных после очистки.

Пример кода:

```python
df.replace([np.inf, -np.inf], np.nan, inplace=True)
df.dropna(inplace=True)
```
