При проектировании современных систем обнаружения вторжений (IDS) на базе машинного обучения качество входных данных определяет итоговую безопасность периметра. Набор данных **CSE-CIC-IDS2018**, несмотря на свой статус эталона, содержит ряд критических дефектов и аномалий, возникших на этапе перехвата трафика и его агрегации утилитой CICFlowMeter.

Если передать эти данные в алгоритм Random Forest без предварительной очистки, математический аппарат модели будет обучаться на техническом шуме и артефактах сбора данных, игнорируя реальные закономерности сетевых атак. # Модуль 1. Дублирование строк-заголовков

## 1. Суть и природа проблемы

Данная аномалия возникла на этапе объединения нескольких CSV-файлов в единый датасет. При склейке файлов строка заголовков могла многократно попадать внутрь набора данных.

В результате среди числовых записей появляются строки вида:

```text
Dst Port,Protocol,Timestamp,Flow Duration,...,Label
```

Для библиотек анализа данных это приводит к нарушению структуры столбцов. Вместо числового типа данные начинают интерпретироваться как текстовые, что вызывает ошибки обучения моделей машинного обучения.

## 2. Полный рабочий код решения

```python
import pandas as pd

print(f"Размерность ДО очистки заголовков: {df.shape}")

df = df[df['Label'] != 'Label']

print(f"Размерность ПОСЛЕ очистки заголовков: {df.shape}")
```

## 3. Подробное объяснение работы кода

* `df['Label'] != 'Label'` создает логическую маску.
* Для каждой строки проверяется содержимое столбца `Label`.
* Если в строке находится повторный заголовок, условие возвращает `False`.
* Конструкция `df[...]` оставляет только строки с `True`.
* В результате все повторные заголовки удаляются.

# Модуль 2. Текстовые маркеры бесконечности

## 1. Суть и природа проблемы

Некоторые характеристики трафика вычисляются как отношение объема данных к длительности соединения.

Если длительность соединения равна нулю:

```text
скорость = объем / 0
```

результатом становится бесконечность.

В датасете такие значения записаны строками:

```text
Infinity
-Infinity
```

Для алгоритмов машинного обучения это является недопустимым типом данных.

## 2. Полный рабочий код решения

```python
import numpy as np
import pandas as pd

inf_markers = ["Infinity", "-Infinity", "inf", "-inf"]

df.replace(inf_markers, np.nan, inplace=True)

print("Текстовые бесконечности успешно конвертированы в системные пустоты.")
```

## 3. Подробное объяснение работы кода

* `inf_markers` содержит все варианты записи бесконечности.
* `replace()` выполняет замену по всему датафрейму.
* Каждое найденное значение заменяется на `np.nan`.
* `inplace=True` выполняет изменение непосредственно в исходном объекте.

# Модуль 3. Текстовые значения NaN

## 1. Суть и природа проблемы

Во время атак и сетевых сбоев некоторые параметры не удается вычислить.

Вместо пустого значения программа записывает строки:

```text
NaN
nan
```

С точки зрения Python это обычный текст, а не специальный маркер отсутствующих данных.

## 2. Полный рабочий код решения

```python
import numpy as np
import pandas as pd

nan_markers = ["NaN", "nan", " ", ""]

df.replace(nan_markers, np.nan, inplace=True)

print("Все текстовые маркеры пропусков приведены к единому системному типу np.nan.")
```

## 3. Подробное объяснение работы кода

* Создается список всех вариантов записи пропусков.
* Метод `replace()` заменяет найденные строки на `np.nan`.
* После этого все отсутствующие значения получают единый формат представления.

# Модуль 4. Нарушение типов данных

## 1. Суть и природа проблемы

Если в числовый столбец попадает хотя бы одно текстовое значение, Pandas автоматически переводит весь столбец в тип `object`.

В результате признаки перестают быть числовыми и становятся несовместимыми с алгоритмами машинного обучения.

## 2. Полный рабочий код решения

```python
import pandas as pd
import numpy as np

service_fields = ['Timestamp', 'Label']
target_numeric_cols = df.columns.drop(service_fields)

for column_name in target_numeric_cols:
    df[column_name] = pd.to_numeric(
        df[column_name],
        errors='coerce'
    )

df.dropna(subset=target_numeric_cols, inplace=True)

print("Типы данных успешно восстановлены.")
```

## 3. Подробное объяснение работы кода

* Исключаются служебные столбцы.
* Все остальные признаки конвертируются в числовой формат.
* При невозможности преобразования создается `NaN`.
* Затем строки с ошибочными значениями удаляются.

---

# Модуль 5. Неверный тип данных порта назначения

## 1. Суть и природа проблемы

После массовой конвертации числовых признаков столбец `Dst Port` получает тип `float64`.

Порты сети являются целыми числами и не должны храниться как вещественные значения.

## 2. Полный рабочий код решения

```python
import numpy as np

df['Dst Port'] = df['Dst Port'].astype(np.int32)

print("Признак Dst Port успешно приведен к типу int32.")
```

## 3. Подробное объяснение работы кода

* Метод `.astype()` изменяет тип данных столбца.
* Значения вида `80.0` преобразуются в `80`.
* Снижается объем памяти и повышается корректность интерпретации признака.

# Модуль 6. Полные дубликаты сетевых сессий

## 1. Суть и природа проблемы

Из-за особенностей захвата трафика одна и та же сессия могла быть записана несколько раз.

Это приводит к эффекту Data Leakage и искусственному завышению точности модели.

## 2. Полный рабочий код решения

```python
import pandas as pd

print(f"Количество строк ДО удаления: {df.shape[0]}")

df.drop_duplicates(inplace=True)

print(f"Количество строк ПОСЛЕ удаления: {df.shape[0]}")
```

## 3. Подробное объяснение работы кода

* Метод `drop_duplicates()` сравнивает все строки.
* Если обнаруживаются полные совпадения, лишние копии удаляются.
* Остается только первое вхождение записи.

# Модуль 7. Ошибки разметки классов

## 1. Суть и природа проблемы

В разных файлах одна и та же атака может иметь различные написания.

Например:

```text
Infiltration
Infilteration
```

Для модели это два разных класса.

## 2. Полный рабочий код решения

```python
import pandas as pd

df['Label'] = df['Label'].str.strip()

linguistic_fixes = {
    'Infilteration': 'Infiltration',
    'Brute Force -Web': 'Brute Force Web',
    'Brute Force -XSS': 'Brute Force XSS'
}

df['Label'] = df['Label'].replace(linguistic_fixes)

print(df['Label'].unique())
```

## 3. Подробное объяснение работы кода

* `strip()` удаляет скрытые пробелы.
* Создается словарь исправлений.
* Метод `replace()` приводит названия классов к единому стандарту.

# Модуль 8. Различные форматы временных меток

## 1. Суть и природа проблемы

Разные серверы могут сохранять даты в различных форматах.

Пример:

```text
2024-01-15
15/01/2024
01/15/2024
```

Это затрудняет обработку времени.

## 2. Полный рабочий код решения

```python
import pandas as pd

df['Timestamp'] = pd.to_datetime(
    df['Timestamp'],
    errors='coerce'
)

df.dropna(subset=['Timestamp'], inplace=True)

print("Все временные метки успешно унифицированы.")
```

## 3. Подробное объяснение работы кода

* `to_datetime()` преобразует строки в формат даты.
* Ошибочные записи превращаются в `NaT`.
* Далее они удаляются методом `dropna()`.

# Модуль 9. Экстремальные выбросы Flow Duration

## 1. Суть и природа проблемы

Некоторые записи содержат аномально большие значения длительности соединений.

Такие выбросы искажают статистику и ухудшают обучение модели.

## 2. Полный рабочий код решения

```python
import pandas as pd

duration_limit = df['Flow Duration'].quantile(0.99)

print(
    f"Порог 99-го процентиля: "
    f"{duration_limit}"
)

df = df[
    df['Flow Duration'] <= duration_limit
]

print("Экстремальные выбросы успешно удалены.")
```

## 3. Подробное объяснение работы кода

* Вычисляется 99-й процентиль.
* Все значения выше найденного порога считаются выбросами.
* Такие записи исключаются из набора данных.

# Модуль 10. Константные признаки

## 1. Суть и природа проблемы

Некоторые признаки содержат одно и то же значение во всех строках.

Например:

```text
0
0
0
0
0
```

Такие признаки не несут полезной информации.

## 2. Полный рабочий код решения

```python
import pandas as pd

constant_cols = [
    col
    for col in target_numeric_cols
    if col in df.columns
    and df[col].nunique() <= 1
]

print(
    f"Обнаружено константных признаков: "
    f"{len(constant_cols)}"
)

print(constant_cols)

df.drop(
    columns=constant_cols,
    inplace=True
)

print(
    f"Финальная размерность: "
    f"{df.shape}"
)
```

## 3. Подробное объяснение работы кода

* `nunique()` подсчитывает количество уникальных значений.
* Если оно меньше либо равно единице, признак считается константой.
* Такие столбцы удаляются методом `drop()`.
* Размерность данных уменьшается, а обучение модели ускоряется.
