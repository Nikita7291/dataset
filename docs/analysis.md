# Технический анализ качества данных: Пошаговый разбор 8 критических дефектов датасета CSE-CIC-IDS2018 по методологии ИСП РАН

При проектировании систем обнаружения вторжений (IDS) на базе машинного обучения качество входных данных определяет стабильность и обобщающую способность моделей. Набор данных **CSE-CIC-IDS2018**, несмотря на широкое применение, содержит ряд глубоких дефектов и физических аномалий, возникших на этапе перехвата трафика и его агрегации утилитой `CICFlowMeter`.

Если передать эти данные в классификатор (например, `Random Forest`) без специализированной обработки, алгоритм обучится на техническом шуме и артефактах сбора данных, полностью потеряв способность обнаруживать реальные угрозы в производственной сети. # Модуль 1. Утечка данных (Data Leakage) и коллинеарные дубликаты столбцов

## 1. Суть и природа проблемы

Аномалия заключается в наличии статических инфраструктурных параметров испытательного стенда: фиксированных IP-адресов атакующих и жертв, портов источников, идентификаторов потоков и точного времени проведения тестов (`Timestamp`, `Src IP`, `Dst IP`, `Source IP`, `Destination IP`, `Flow ID`, `Src Port`).

Деревья решений мгновенно привязываются к этим маркерам, выдавая ложную точность на тестовой выборке лаборатории, но полностью теряя способность к классификации в реальной сети с другими IP-адресами.

Дополнительно из-за аппаратных сбоев парсера возникли скрытые коллинеарные дубликаты признаков с суффиксами `.1` и `.2`.

## 2. Полный рабочий код решения

```python
import pandas as pd
import numpy as np
import glob
import os

input_dir = r"C:\Users\user\CSE-CIC-IDS2018_ML"
source_files = glob.glob(os.path.join(input_dir, "*.csv"))

output_dir = os.path.join(input_dir, "final_cleaned_datasets")
os.makedirs(output_dir, exist_ok=True)

leakage_cols = [
    'Timestamp',
    'Src IP',
    'Dst IP',
    'Source IP',
    'Destination IP'
]

for file_path in source_files:
    file_name = os.path.basename(file_path)

    if file_name.startswith("final_clean_"):
        continue

    preview = pd.read_csv(file_path, nrows=5)

    cols_to_drop = [
        col for col in preview.columns
        if col.endswith('.1') or col.endswith('.2')
    ]

    cols_to_drop.extend(
        [col for col in leakage_cols if col in preview.columns]
    )
```

## 3. Подробное объяснение работы кода

- Импортируются библиотеки для работы с файлами и таблицами.
- Получается список всех CSV-файлов датасета.
- Создается каталог для очищенных данных.
- Формируется список признаков, способных вызвать Data Leakage.
- Первые строки каждого файла используются для анализа структуры колонок.
- Выявляются дублированные столбцы с суффиксами `.1` и `.2`.
- Формируется итоговый список признаков для удаления.

# Модуль 2. Вшитые строки-заголовки и синтаксический мусор целевой переменной

## 1. Суть и природа проблемы

Во время объединения CSV-файлов строки заголовков повторно попали внутрь датасета.

В результате среди числовых данных появились текстовые строки с названиями столбцов.

Кроме того, поле `Label` содержит скрытые пробелы и орфографические ошибки, что искусственно увеличивает число классов.

## 2. Полный рабочий код решения

```python
for chunk in pd.read_csv(
        file_path,
        chunksize=300000,
        low_memory=False,
        engine='c'):

    if 'Label' in chunk.columns:
        chunk = chunk[
            chunk['Label'].astype(str).str.lower() != 'label'
        ]

        chunk['Label'] = (
            chunk['Label']
            .astype(str)
            .str.strip()
        )

    if 'Flow ID' in chunk.columns:
        chunk = chunk.dropna(subset=['Flow ID'])

        chunk = chunk[
            chunk['Flow ID']
            .astype(str)
            .str.strip() != ''
        ]
```

## 3. Подробное объяснение работы кода

- Данные читаются частями для экономии памяти.
- Встроенные заголовки удаляются через логическую фильтрацию.
- Метод `strip()` очищает текст от лишних пробелов.
- Проверяется корректность идентификаторов потоков.
- Поврежденные записи удаляются.

# Модуль 3. Строковый мусор Infinity и NaN

## 1. Суть и природа проблемы

В скоростных признаках встречаются значения:

```text
Infinity
-Infinity
NaN
```

Они возникают при делении объема трафика на нулевую длительность соединения.

Подобные значения нарушают работу математических алгоритмов анализа данных.

## 2. Полный рабочий код решения

```python
for col in numeric_cols:
    if col in chunk.columns:
        chunk[col] = pd.to_numeric(
            chunk[col],
            errors='coerce'
        )

chunk = chunk.replace(
    [np.inf, -np.inf],
    np.nan
)

available_numeric = [
    c for c in numeric_cols
    if c in chunk.columns
]

chunk = chunk.dropna(
    subset=available_numeric
)
```

## 3. Подробное объяснение работы кода

- Все числовые признаки принудительно конвертируются в числа.
- Ошибочные строки заменяются на `NaN`.
- Вещественные бесконечности заменяются на пропуски.
- Поврежденные строки удаляются.

# Модуль 4. Баг FIN и фрагментация TCP-сессий

## 1. Суть и природа проблемы

При обнаружении первого пакета с флагом `FIN` утилита преждевременно завершает обработку соединения.

Последующие пакеты подтверждения формируют новые ложные сессии с нулевой полезной нагрузкой.

## 2. Полный рабочий код решения

```python
bad_mask = pd.Series(
    False,
    index=chunk.index
)

if (
    'Tot Fwd Pkts' in chunk.columns
    and
    'TotLen Fwd Pkts' in chunk.columns
):
    bad_mask |= (
        (chunk['Tot Fwd Pkts'] > 0)
        &
        (chunk['TotLen Fwd Pkts'] == 0)
    )

if (
    'Tot Bwd Pkts' in chunk.columns
    and
    'TotLen Bwd Pkts' in chunk.columns
):
    bad_mask |= (
        (chunk['Tot Bwd Pkts'] > 0)
        &
        (chunk['TotLen Bwd Pkts'] == 0)
    )
```

## 3. Подробное объяснение работы кода

- Создается булева маска аномалий.
- Выявляются сессии с пакетами, но без полезных данных.
- Такие записи считаются артефактами обработки FIN.
- Все найденные строки подлежат удалению.

# Модуль 5. Баг RST и переполнение счетчиков

## 1. Суть и природа проблемы

Игнорирование флага `RST` приводит к зависанию сессий в памяти и смешиванию разных соединений.

Это вызывает отрицательные значения длительности, портов и межпакетных интервалов.

## 2. Полный рабочий код решения

```python
if 'Dst Port' in chunk.columns:
    bad_mask |= (chunk['Dst Port'] < 0)

if 'Flow Duration' in chunk.columns:
    bad_mask |= (chunk['Flow Duration'] < 0)

iat_cols = [
    c for c in
    ['Flow IAT Min', 'Fwd IAT Min', 'Bwd IAT Min']
    if c in chunk.columns
]

for col in iat_cols:
    bad_mask |= (chunk[col] < 0)

chunk_clean = chunk[~bad_mask]
```

## 3. Подробное объяснение работы кода

- Отрицательные значения считаются физически невозможными.
- Проверяются порты, длительности и интервалы.
- Все строки с признаками переполнения удаляются.
- Формируется очищенный набор данных.

if is_first_chunk:

# Модуль 8. Баг Average Packet Size и константные признаки

## 1. Суть и природа проблемы

Первый пакет потока учитывается дважды, что искажает расчет среднего размера пакета.

Дополнительно присутствуют константные признаки, содержащие одинаковые значения во всех строках.

## 2. Полный рабочий код решения

```python
constant_per_file = {}

for FILE in ALL_FILES:

    filepath = os.path.join(FOLDER, FILE)

    df = pd.read_csv(filepath)

    num_cols = df.select_dtypes(
        include=np.number
    ).columns

    num_constant = set(
        c for c in num_cols
        if df[c].nunique() <= 1
    )

    all_constant = num_constant

    constant_per_file[FILE] = all_constant

all_constant_sets = list(
    constant_per_file.values()
)

common_constant = all_constant_sets[0]

for s in all_constant_sets[1:]:
    common_constant = common_constant & s

ALL_TO_REMOVE = (
    set(ALWAYS_REMOVE)
    |
    common_constant
)
```

## 3. Подробное объяснение работы кода

- Выполняется анализ всех файлов датасета.
- Выявляются признаки с нулевой дисперсией.
- Определяются общие константные столбцы.
- Формируется единый список удаления.
- Исключаются признаки, не несущие полезной информации.
