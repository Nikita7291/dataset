# Комплексный технический аудит: Детальный разбор 10 критических ошибок датасета CSE-CIC-IDS2018

При проектировании современных систем обнаружения вторжений (IDS) на базе машинного обучения ключевым фактором является качество входных данных. Набор данных **CSE-CIC-IDS2018**, несмотря на свою эталонность, содержит ряд фундаментальных дефектов и аномалий, возникших на этапе перехвата трафика и его агрегации утилитой CICFlowMeter. 

Если передать эти данные в алгоритм Random Forest «как есть», математический аппарат модели будет переобучаться на техническом мусоре, полностью игнорируя реальные сетевые угрозы. Ниже представлен глубокий анализ 10 выявленных нами проблем и монолитный production-код их решения.

---

## 🛑 Проблема №0: Экстремальный дисбаланс и дефицит опасного трафика

Это главная концептуальная проблема датасета. Сетевой трафик в реальной инфраструктуре и в файлах CSE-CIC-IDS2018 характеризуется подавляющим преобладанием легитимной активности (`Benign`) — **от 85% до 99.5%** в зависимости от конкретного дня съема. Опасные инциденты (например, атаки типа `Infiltration` или `DoS-Slowloris`) исчисляются долями процента или вовсе единичными строками на миллион записей.

### Почему это уничтожает Random Forest:
Математическая функция потерь алгоритма Random Forest оптимизирует метрику общего количества правильных ответов (**Accuracy**). В условиях, когда 98% данных принадлежат к классу `Benign`, модель моментально находит тривиальное решение: она учится всегда и при любых условиях выдавать вердикт «Нормальный трафик». При этом общая точность модели составит великолепные **98%**, что может ввести в заблуждение неопытного исследователя. Однако на практике полнота обнаружения атак (**Recall**) будет равна нулю — система пропустит 100% реальных кибератак.

### Наше инженерное решение:
В нашем коде мы применили двухэтапный метод стабилизации:
1. **Программный Downsampling (Уменьшение нормы):** Мы искусственно сокращаем избыточный объем безопасных сессий `Benign`, случайным образом отбирая подвыборку, сопоставимую с суммарным количеством опасных инцидентов.
2. **Алгоритмическая балансировка весов дерева:** При инициализации классификатора мы принудительно активируем параметр `class_weight='balanced'`. Это заставляет алгоритм рассчитывать штрафы за ошибку обратно пропорционально частоте класса. Ошибка на одной строке редкой атаки наказывает модель в сотни раз сильнее, чем ошибка на строке фонового трафика.

## 🛠️ ПОЛНЫЙ PRODUCTION-КОД ОЧИСТКИ ДАТАСЕТА (Скрипт предобработки)

Ниже представлен полный монолитный код на Python, который последовательно устраняет все 10 дефектов датасета и решает проблему дисбаланса классов.

```python
import pandas as pd
import numpy as np
from sklearn.utils import resample

def execute_complete_pipeline_cleaning(file_path):
    print("==================================================================")
    print("[СТАРТ] Инициализация конвейера комплексной очистки данных...")
    print("==================================================================")
    
    # Шаг 0: Загрузка данных. Отключаем low_memory, чтобы избежать предупреждений о типах
    print("[ШАГ 0] Чтение исходного CSV-файла в оперативную память...")
    df = pd.read_csv(file_path, low_memory=False)
    initial_rows, initial_cols = df.shape
    print(f"[ИНФО] Исходная размерность датафрейма: {initial_rows} строк, {initial_cols} признаков.")
    
    # -------------------------------------------------------------------------
    # ПРОБЛЕМА 1: Дублирование строк-заголовков (Технический брак склейки файлов)
    # Объяснение: При объединении логов за день имена колонок повторно вклеивались внутрь данных.
    print("[ШАГ 1] Удаление внутренних дубликатов заголовков колонок...")
    df = df[df['Label'] != 'Label']
    
    # -------------------------------------------------------------------------
    # ПРОБЛЕМА 2 & 3: Текстовые маркеры бесконечности и пропусков ('Infinity', 'NaN')
    # Объяснение: Скорость вычислялась делением на время. При длительности сессии 0 мс
    # парсер CICFlowMeter записывал строки "Infinity" и "NaN", ломая числовой тип столбца.
    print("[ШАГ 2] Конвертация текстовых маркеров 'Infinity' и 'NaN' в системные NaN...")
    corrupted_markers = ["Infinity", "-Infinity", "inf", "-inf", "NaN", "nan"]
    df.replace(corrupted_markers, np.nan, inplace=True)
    
    # -------------------------------------------------------------------------
    # ПРОБЛЕМА 4: Нарушение типов данных (Столбцы распознаны как текстовый тип 'object')
    # Объяснение: Из-за строкового мусора Pandas считал все числовые метрики строками.
    # Исправление: Принудительный перевод всех сетевых метрик во float64.
    print("[ШАГ 3] Принудительное приведение типов сетевых признаков к float64...")
    service_fields = ['Timestamp', 'Label']
    target_numeric_cols = df.columns.drop(service_fields)
    
    for column_name in target_numeric_cols:
        # errors='coerce' автоматически превратит любые остаточные строковые ошибки в np.nan
        df[column_name] = pd.to_numeric(df[column_name], errors='coerce')
    
    # Каскадное удаление всех строк, где после конвертации образовались NaN значения
    print("[ШАГ 4] Каскадное удаление строк, содержащих пропуски (NaN)...")
    df.dropna(subset=target_numeric_cols, inplace=True)
    
    # -------------------------------------------------------------------------
    # ПРОБЛЕМА 5: Некорректный вещественный тип в поле сетевых портов (Dst Port)
    # Объяснение: Сетевой порт — это дискретное целое число (0-65535). Тип float здесь недопустим.
    print("[ШАГ 5] Оптимизация типов данных: приведение 'Dst Port' к целочисленному int32...")
    df['Dst Port'] = df['Dst Port'].astype(np.int32)
    
    # -------------------------------------------------------------------------
    # ПРОБЛЕМА 6: Полные дубликаты сетевых сессий (Сетевое эхо из-за зеркалирования)
    # Объяснение: Ошибки настройки SPAN/TAP портов приводили к двойному захвату сессий.
    # Это вызывает Data Leakage (утечку данных) между обучающей и тестовой выборками.
    print("[ШАГ 6] Поиск и жесткое удаление полных дубликатов сетевых сессий...")
    df.drop_duplicates(inplace=True)
    
    # -------------------------------------------------------------------------
    # ПРОБЛЕМА 7: Лингвистические опечатки и дублирование классов в поле разметки 'Label'
    # Объяснение: Наличие классов 'Infilteration' и 'Infiltration' размывает фокус обучения модели.
    print("[ШАГ 7] Стандартизация текстовых меток классов и устранение опечаток...")
    df['Label'] = df['Label'].str.strip() # Удаляем невидимые пробелы на концах строк
    class_mapping_fixes = {
        'Infilteration': 'Infiltration',
        'Brute Force -Web': 'Brute Force Web',
        'Brute Force -XSS': 'Brute Force XSS'
    }
    df['Label'] = df['Label'].replace(class_mapping_fixes)
    
    # -------------------------------------------------------------------------
    # ПРОБЛЕМА 8: Разнородные и ломаные форматы временных меток (Timestamp)
    # Объяснение: В зависимости от дня недели даты записывались то в ISO, то в американском формате.
    print("[ШАГ 8] Унификация временных меток и очистка поврежденных дат...")
    df['Timestamp'] = pd.to_datetime(df['Timestamp'], errors='coerce')
    df.dropna(subset=['Timestamp'], inplace=True)
    
    # -------------------------------------------------------------------------
    # ПРОБЛЕМА 9: Экстремальные выбросы счетчиков длительности сессии (Flow Duration)
    # Объяснение: Сбои таймеров ядер ОС приводили к генерации нереального времени сессий.
    # Исправление: Фильтрация аномалий по строгому верхнему 99-му квантилю.
    print("[ШАГ 9] Фильтрация экстремальных выбросов длительности потока (Flow Duration)...")
    duration_upper_bound = df['Flow Duration'].quantile(0.99)
    df = df[df['Flow Duration'] <= duration_upper_bound]
    
    # -------------------------------------------------------------------------
    # ПРОБЛЕМА 10: Избыточные признаки с нулевой дисперсией (Zero Variance Columns)
    # Объяснение: Столбцы, содержащие константы (например, только нули), бесполезны для обучения леса.
    print("[ШАГ 10] Идентификация и исключение признаков с нулевой дисперсией...")
    columns_to_analyze = [c for c in target_numeric_cols if c in df.columns]
    zero_variance_cols = [col for col in columns_to_analyze if df[col].nunique() <= 1]
    df.drop(columns=zero_variance_cols, inplace=True)
    print(f"[ИНФО] Удалено константных признаков: {len(zero_variance_cols)} -> {zero_variance_cols}")
    
    rows_after_math_cleaning = df.shape[0]
    print(f"[ИНФО] Строк после математической очистки: {rows_after_math_cleaning} (Удалено: {initial_rows - rows_after_math_cleaning})")
    
    # -------------------------------------------------------------------------
    # РЕШЕНИЕ ПРОБЛЕМЫ ДИСБАЛАНСА КЛАССОВ (Умный Downsampling мажоритарного класса Benign)
    print("[БАЛАНСИРОВКА] Запуск процедуры выравнивания долей классов...")
    df_benign_subset = df[df['Label'] == 'Benign']
    df_attacks_subset = df[df['Label'] != 'Benign']
    
    total_attacks_volume = df_attacks_subset.shape[0]
    # Настраиваем пропорцию 1.5 к 1 (Benign к атакам), чтобы сохранить репрезентативность фона
    optimized_benign_samples = int(total_attacks_volume * 1.5)
    
    if df_benign_subset.shape[0] > optimized_benign_samples:
        print(f"[БАЛАНС] Сокращаем объем Benign с {df_benign_subset.shape[0]} до {optimized_benign_samples} строк...")
        df_benign_downsampled = resample(df_benign_subset, 
                                         replace=False, 
                                         n_samples=optimized_benign_samples, 
                                         random_state=42)
        df_final_balanced = pd.concat([df_benign_downsampled, df_attacks_subset])
    else:
        df_final_balanced = df.copy()
        
    print("==================================================================")
    print(f"[ФИНИШ] Конвейер успешно завершен. Итоговая размерность: {df_final_balanced.shape}")
    print("Распределение классов в очищенном датасете:")
    print(df_final_balanced['Label'].value_counts())
    print("==================================================================")
    
    return df_final_balanced
